supergraph

One file. Every module. The whole monorepo.

supergraph analyzes your monorepo's audit data and generates a single interactive HTML file — a physics-simulated graph of every module, every import, and every cross-package edge. Drop it into any AI context window and the agent instantly understands your entire codebase.

$ curl -fsSL https://raw.githubusercontent.com/bravenewxyz/supergraph/main/bin/supergraph -o /usr/local/bin/supergraph && chmod +x /usr/local/bin/supergraph

installation

Compile once.
Run anywhere.

supergraph compiles to a single native binary via Bun. No Node.js, no npm, no runtime. Copy it to your PATH and it's ready.

01
Clone the repo
Get the source code.
shell 
git clone https://github.com/bravenewxyz/supergraph
cd supergraph
02
Compile the binary
Bun compiles the TypeScript source to a self-contained native executable. Target arm64 macOS or adjust for your platform.
shell — macOS arm64 
bun build --compile --target=bun-darwin-arm64 src/index.ts --outfile bin/supergraph
shell — macOS x64 
bun build --compile --target=bun-darwin-x64 src/index.ts --outfile bin/supergraph
shell — Linux x64 
bun build --compile --target=bun-linux-x64 src/index.ts --outfile bin/supergraph
03
Add to PATH
Copy the compiled binary to a directory on your PATH so you can run it from any project.
shell 
cp bin/supergraph ~/.local/bin/supergraph
Note: make sure ~/.local/bin is on your $PATH. Add export PATH="$HOME/.local/bin:$PATH" to your shell profile if needed.
usage

Run it.
Open the file.

supergraph reads the audit/ directory generated by your devtools pipeline and produces audit/supergraph.html. The audit data must already exist.

01
Navigate to your monorepo
supergraph looks for an audit/ directory in the current working directory.
shell 
cd your-monorepo
02
Run supergraph
Generates audit/supergraph.html from the audit JSON data. Takes a few seconds for large monorepos.
shell 
supergraph
shell — custom root path 
supergraph --root ./path/to/repo
03
Open the output
The graph is a fully self-contained HTML file. Open it in any browser. No server, no build step, no internet required.
shell 
open audit/supergraph.html
Prerequisite: the audit/ directory must already exist with JSON maps of your packages. supergraph is part of a devtools pipeline — it reads pre-generated data, it does not analyze source files itself.
for AI agents

One file is all
an agent needs.

The original motivation for supergraph: give any AI coding agent complete, structured knowledge of a large monorepo in a single context attachment.

01 — single-file context
One attachment, full picture
The generated HTML encodes the entire module graph as structured data. Paste it into an agent's context — or reference it directly — and the agent has a complete map of every module path, every export, every import edge.
02 — full symbol graph
Every exported symbol, annotated
Each node includes all exported functions, types, constants, and classes from that module. An agent can answer "where is X defined?" or "what does package Y export?" without reading a single source file.
03 — cross-package edges
Dependency relationships made explicit
Cross-package imports are the hardest relationships to infer from raw source. supergraph makes them first-class: every inter-package edge is labeled, directional, and queryable. Agents stop confusing internal and external dependencies.
04 — issue annotations
Problems surface automatically
Circular dependencies, missing re-exports, and broken import paths are annotated directly on affected nodes. An agent reading the graph immediately knows where the codebase has structural debt — no grep required.

Large monorepos routinely exceed the context window of any AI model when loaded naively. supergraph is the answer to that problem. Instead of hoping the agent can infer structure from a handful of files, you give it a pre-analyzed, structured, machine-readable map of every relationship in the codebase. The agent knows the architecture before writing a single line of code.

keyboard shortcuts

Navigate
without a mouse.

The generated graph supports keyboard shortcuts for the most common interactions. Works when the canvas has focus.

key action notes
p Toggle physics simulation Pause or resume the force-directed layout. Useful when nodes have settled and you want to lock positions.
l Toggle module labels Show or hide path labels on all nodes. Labels still appear on hover when hidden. Helps with dense graphs.
Escape Close detail panel Dismisses the node detail panel showing module info, exports, and import edges.
scroll Zoom in / out Mouse wheel or trackpad scroll zooms the canvas around the cursor position.
drag Pan canvas Click and drag on the background to pan the view. Drag a node to pin its position.
